# Nifty Index Analysis — MCP Server

A Model Context Protocol (MCP) server that exposes Nifty index historical data, SIP backtesting, technical indicators, and market analysis as AI-callable tools.

## Project Structure

```
mcp_nifty/
├── __init__.py          # Package marker
├── server.py            # FastMCP server entry point (registers all tools)
├── models.py            # Pydantic models for input validation & output schemas
├── helpers.py           # Data loading, API calls, calculation utilities
└── tools/
    ├── __init__.py
    ├── data.py          # Data retrieval tools
    ├── analysis.py      # Drop stats, CAGR, compare, top movers
    ├── backtest.py      # SIP backtesting tools
    └── indicators.py    # Technical indicators & strategy signals
```

## Tools (17 total)

### Data Retrieval
| Tool | Description |
|------|-------------|
| `list_indices` | List all indices with metadata, optionally filter by type |
| `get_index_summary` | Summary stats: date range, prices, avg return, worst/best day |
| `get_index_data` | Historical daily price data with date filtering |
| `fetch_live_price` | Fetch latest price from NSE API |

### Analysis
| Tool | Description |
|------|-------------|
| `get_drop_stats` | Count days with drops ≥ threshold, avg/worst drop |
| `get_cagr` | CAGR between any two dates |
| `compare_indices` | Compare multiple indices by CAGR, volatility, worst drop |
| `get_top_movers` | Top 5 gainers/losers across time windows |

### Backtesting
| Tool | Description |
|------|-------------|
| `backtest_sip` | Fixed-day monthly SIP with XIRR calculation |
| `backtest_dip_sip` | Deploy cash only on dip days (threshold-based) |
| `find_best_sip_day` | Optimal day of month (1-28) by XIRR |

### Technical Indicators & Strategies
| Tool | Description |
|------|-------------|
| `get_technical_indicators` | SMA, EMA, RSI, Bollinger Bands, MACD |
| `get_moving_average_signals` | Golden/death cross detection with trend |
| `get_drawdown_analysis` | Max drawdown, ATH distance, recovery periods |
| `get_momentum_ranking` | Rank all indices by momentum |
| `get_volatility_analysis` | Rolling vol, regime detection, percentile |
| `get_mean_reversion_signals` | Z-score based overbought/oversold signals |

## Setup

```bash
# Install dependencies
pip install "mcp[cli]" numpy pandas scipy requests pydantic

# Run the server
python -m mcp_nifty.server
```

## VS Code / Claude Desktop Integration

Add to your MCP settings (`settings.json` or `claude_desktop_config.json`):

```json
{
  "mcpServers": {
    "nifty-index": {
      "command": "python",
      "args": ["-m", "mcp_nifty.server"],
      "cwd": "/path/to/VOLUMES"
    }
  }
}
```

## Data Requirements

The server reads from these JSON files in the parent directory:

- `all_index_data.json` — Full historical data (fetched via `fetch_all_data.py`)
- `all_index_summary.json` — Index metadata
- `top_movers.json` — Pre-computed top movers (generated by `trend.py`)

## Input Validation

All tool inputs are validated via Pydantic models with:
- Type checking and coercion
- Range constraints (e.g., `sip_day` must be 1-28)
- Required vs optional fields with defaults
- Descriptive error messages on validation failure
